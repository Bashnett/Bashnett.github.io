Cicada Write-up(HacktheBox)

Author: Anish Basnet

This writeup covers the exploitation of a vulnerable Windows machine, detailing the steps to discover and exploit misconfigurations to gain unauthorized access.

Step 1: Reconnaissance

The first step involves gathering as much information as possible using tools like nmap for port scanning and enumeration. 


$ nmap -sC -sV -oN initial_scan.txt 10.10.10.10

This scan reveals several open ports, including port 445 (SMB), which will be the target for this attack.

Step 2: Exploiting SMB

With port 445 open, we try to enumerate shares using the enum4linux tool: 


$ enum4linux -a 10.10.10.10

We discover a shared directory with sensitive information. We access this share and find a file containing the username and password for the machine.

"Successful enumeration and exploitation often come down to identifying weakly secured services."

Step 3: Gaining Access

Using the credentials found, we attempt to log into the machine via SMB: 


$ smbclient \\\\10.10.10.10\\share -U user

With access to the machine, we can now escalate privileges and gain full control of the system.

Step 4: Privilege Escalation

After gaining initial access, we run privilege escalation scripts such as winPEAS to identify potential vectors for privilege escalation. A misconfigured service is identified, allowing us to escalate privileges to the administrator account. 


$ winPEAS.exe

Conclusion

This writeup demonstrates the importance of securing services like SMB and ensuring proper access controls on shared directories. Enumeration is key to exploiting vulnerabilities, and the entire process highlights how attackers can chain small misconfigurations into a full compromise.
